Getting Started - Mixed Mode - Browserify Seed

Why use the Browserify seed? When using Famo.us, your project will often require a long list of dependencies all declared at the top of your file. In order to manage these dependencies more easily, the Browserify seed includes helpful scripts to run, gather, and bundle all of your dependencies using Browserify.

Here’s how it works: At the top of your root JavaScript file (check your package.json to find this -by default it’s the index.js file) simply declare your dependencies using the same ‘require’ notation as you would with Node.js.

When you run an ‘npm build’ or an ‘npm start-dev,’ Browserify will look at your root file, gather all the required dependencies and then compile them into a single bundle.js file. Now, instead of linking to multiple scripts in your index.html file, you can include your entire project in a single script tag.

The Browserify seed keeps everything tidy in the browser and makes sure your code stays modular behind the scenes!
