THE IMAGE GRABBER


Problem solved – Takes a SKU, finds its image location and then imports it for easy use with reports.



ABOUT: Before I built this tool there was no quick way to view an image for a given SKU. Image requests would be sent to our web team who would look for the file scattered among thousands of folders on the server and then send it back via email. Frustrated with this lengthy process I sat down with our web team and mapped out where and how each image was saved on the server. Using this info I wrote a function that builds an image path based on the SKU and when it was uploaded. I then added a button that uses this path to import the image in the adjacent row. Later I added two more VBAs to select and delete all the photos for easier use with reports.

FORMAT: First SKUs are pasted into the grey column under ENTER SKU HERE: In column C an image path is displayed so the user can manually grab an image or find the alt images for a given SKU. The path is built using a VLOOKUP to an external file that lists the upload date for each SKU. Using this date the function concatenates a path in this format:

Z:\snap\(year uploaded) + -shared\ (upload date)\COMPLETE\(SKU without spaces) + _ + (SKU color) +.jpg

In the next column a “GRAB!” button is assigned to a VBA that uses the path in column C to import the image. Below this button conditional formatting checks if the SKU format is valid, and prompts an error if not. Additionally if the image is not found when the button is pressed an alert box will notify the user.

Column F is where the image appears after the grab button is pressed. The image height is set to the row height and it is imported as a linked image. This allows you to resize the image without losing image quality. Two other buttons assigned to VBAs were added to help clear and select images. This was added since you cannot highlight multiple images using the rectangular Marquee as you would with text or tables.
